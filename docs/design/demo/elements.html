Why only 13 elements here — and how to add one

This page mirrors contract/elements.js, which deliberately locks only atomic, high-reuse, low-variance roles — not all 125. 13 is not a target to grow; a role belongs here only when it stays hand-typed identically everywhere and a tag would remove that drift. Composites (table, modal, kanban, cmdk, every domain pattern) are intentionally excluded — their DOM legitimately varies per page, so they stay raw markup (optionally nesting these locked tags inside).

To extract a new atomic — 3-question test (all must be yes)
  1. Atomic / near-atomic? Few sub-parts, not a many-slot composite.
  2. Reused across ≥2 domains/pages? Not local to one screen's content.
  3. Fixed content shape? Same handful of slots every time, not free-form children.

Any no → leave it raw. All yes → add the tag to elements.js emitting the exact same .fg-* classes/tokens the raw role already uses (no new token or visual behavior), add a compare block below, and re-verify contract-tests.html all-green. Full rule: SPEC.md § Web Component companion layer.
